## 二、项目愿景

让中小团队的测试工作从「手工重复劳动」升级为「AI驱动的智能质量保障」。

**一句话**：一个对话框搞定从测试设计到执行分析的全流程，让测试工程师把精力花在真正需要判断力的事情上。

---

## 三、项目目标

### 3.1 业务目标

| 目标 | 衡量标准 | 节点 |
|------|---------|------|
| MVP可用 | API测试完整闭环（创建→执行→报告）可演示 | Week 6 |
| AI增强就绪 | 自然语言创建测试+AI失败分析可用 | Week 10 |
| 全栈测试覆盖 | API+Web UI+APP UI+性能压测全部可用 | Week 14 |
| 正式发布 | v1.0 可对外发布/开源 | Week 18 |

### 3.2 技术目标

| 目标 | 衡量标准 |
|------|---------|
| 全Docker化部署 | `docker compose up -d` 一键启动 |
| 前后端分离 | React 18 + FastAPI，RESTful API |
| 数据库高可用 | MySQL 8.0 业务库 + PostgreSQL 14 分析库 |
| 代码质量 | TypeScript 全覆盖，Pylint/ESLint 零告警 |

## 四、项目范围

### 4.1 Phase 0-1 MVP 范围（Week 1-6）

```
✅ 用户认证（注册/登录/JWT）
✅ 用户管理基础CRUD
✅ WebSphere风布局框架（三级菜单+面包屑）
✅ 团队管理
✅ 项目管理
✅ 项目仪表盘
✅ API接口定义管理
✅ 测试用例管理（JSON步骤+断言+数据驱动）
✅ 测试计划管理
✅ API执行引擎（requests + 断言 + 变量替换）
✅ 执行结果详情（请求/响应/日志）
✅ Allure报告+报告查看页
```

### 4.2 明确不在此次范围

```
❌ AI自然语言创建测试（Phase 2）
❌ Web/APP UI自动化（Phase 3）
❌ 性能压测集成（Phase 3）
❌ CI/CD集成插件（Phase 4）
❌ 企业权限系统（Phase 4）
❌ 缺陷管理看板（Phase 4）
❌ 邮件通知系统（Phase 4）
```
