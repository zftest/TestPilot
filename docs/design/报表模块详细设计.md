## 一、报表菜单入口与导航

```
📊 测试报告
  ├── 📋 执行报告        ← 单次执行详情（Allure增强版）
  ├── 📈 质量趋势        ← 跨版本质量数据可视化
  ├── 🚦 质量门禁        ← 发版准入/准出判定
  ├── 🐛 缺陷管理        ← 缺陷列表 + 缺陷看板
  ├── 🔥 性能报告        ← 压测结果汇总 + 趋势对比
  └── 📤 报告导出        ← PDF/HTML/邮件定时推送
```

## 二、执行报告（单次测试详情页）

### 2.1 数据展示内容

| 数据块 | 数据项 | 数据来源 |
|--------|--------|---------|
| **概览头** | 执行ID / 触发方式(手动/CI/CD/定时) / 耗时 / 用例总数/通过/失败/跳过/阻塞 | execution_records表 |
| **通过率环形图** | 通过/失败/跳过/阻塞 四色占比 | test_results表聚合 |
| **失败严重度分布** | Blocker / Critical / Major / Minor / Trivial 环形图 | test_results.severity |
| **用例耗时TOP10** | 最慢10条用例的名称+耗时+状态 | test_results.duration排序 |
| **按模块通过率** | 各模块(登录/订单/支付...)通过率横向柱状图 | test_results.module分组 |
| **失败详情表格** | 用例名/模块/失败类型/断言信息/AI分析/截图 | test_results + ai_analysis_logs |
| **执行日志** | 实时滚动日志流 | WebSocket实时推送 |
| **截图/录屏** | 失败截图、APP端录屏回放 | MinIO对象存储 |

### 2.2 图表规格

| 图表类型 | 展示数据 | 库 | 交互 |
|---------|---------|-----|------|
| **环形图** (doughnut) | 通过率 / 失败严重度 | Chart.js | 点击扇形跳转对应筛选项 |
| **横向柱状图** (horizontal bar) | 模块通过率 / 用例耗时TOP10 | Chart.js | 点击柱体进入该模块用例列表 |
| **趋势折线图** (line) | 批量执行耗时对比（多次执行） | Chart.js | 悬停显示具体数值 |
| **热力图** (heatmap-like) | 用例执行时长分布 | 自定义Canvas | 颜色深浅代表耗时 |

### 2.3 交互功能

- **筛选**：按状态/模块/严重度/用例名称
- **排序**：点击表头排序
- **导出**：单次报告导出PDF
- **分享**：生成只读链接（7天有效）
- **对比**：勾选2条历史执行记录，一键对比diff

## 三、质量趋势页（跨版本可视化）

### 3.1 核心趋势图

| 图名 | 类型 | X轴 | Y轴 | 数据源 |
|------|------|-----|-----|--------|
| **通过率趋势** | 双Y轴折线图 | 日期/版本 | 通过率%(左) + 用例数(右) | test_results按天聚合 |
| **缺陷密度趋势** | 面积图 | 日期/版本 | 缺陷数/1Kloc | defect_table关联commit |
| **用例增长趋势** | 堆叠面积图 | 日期 | API+Web+APP+性能 四色堆叠 | test_cases.created_at |
| **执行耗时趋势** | 折线图 | 日期 | 平均执行耗时(分钟) | test_results.duration |
| **自动化覆盖率** | 进度条+目标线 | 模块 | %覆盖率 | test_plans覆盖率字段 |
| **失败归因分布** | 堆叠柱状图 | 周 | 环境/代码/数据/超时/未知 五色 | ai_analysis_logs.groupby(原因) |

### 3.2 筛选维度

- 时间范围：最近7天/30天/90天/自定义
- 项目：全部/单个
- 测试类型：全部/API/Web UI/APP UI/性能
- 模块：全部/登录/订单/支付/商品...

## 四、质量门禁页（发版准入判定）

### 4.1 门禁规则配置

| 规则项 | 判断条件 | 操作 |
|--------|---------|------|
| 通过率 | ≥ 95% | 不满足=阻塞发版 |
| 失败Blocker数 | = 0 | 不满足=阻塞发版 |
| 自动化覆盖率 | ≥ 80% | 不满足=警告 |
| 新增缺陷数 | ≤ 当前版本5个 | 不满足=警告 |
| 性能回归 | P99 RT ≤ 上版本×1.2 | 不满足=阻塞发版 |
| 用例执行率 | = 100% | 不满足=警告 |

### 4.2 门禁面板UI

展示为一个5行×3列的检查矩阵：
- 左侧：规则名称 + 阈值
- 中间：当前值 + 通过/不通过指示灯
- 右侧：不通过原因 + 建议操作

门禁状态总灯：全部通过=绿色、有警告=黄色、有阻塞=红色

## 五、缺陷管理

### 5.1 缺陷看板（Kanban视图）

| 列 | 状态 | 字段 |
|----|------|------|
| 📝 待处理 | new | 标题/严重度/关联用例/创建人/时间 |
| 🔍 分析中 | analyzing | AI分析结果/指派人 |
| 🛠 修复中 | fixing | 关联commit/修复人 |
| ✅ 待验证 | verifying | 修复版本/验证用例 |
| ✔ 已关闭 | closed | 关闭时间/关闭人 |

缺陷卡片拖拽流转，状态变更自动通知飞书/企微。

### 5.2 缺陷与Jira/TAPD对接

- **自动创建**：门禁不通过时自动在Jira/TAPD创建Issue
- **状态同步**：TestPilot ↔ Jira/TAPD 双向Webhook
- **字段映射**：标题/描述/严重度/指派人/版本/关联用例

## 六、性能报告（压测专项）

### 6.1 单接口压测报告

| 数据块 | 内容 |
|--------|------|
| 概览卡片 | 总请求数/QPS/P50/P95/P99 RT/失败率/并发数 |
| QPS趋势图 | 折线图，X=时间 Y=QPS |
| RT分布图 | 折线图，多线(P50/P95/P99) |
| 百分位表格 | P50/P75/P90/P95/P99/P99.9 |
| 错误分布 | 饼图：连接超时/4xx/5xx/断言失败 |

### 6.2 链路压测报告

| 数据块 | 内容 |
|--------|------|
| 调用链瀑布图 | 每个接口的RT堆叠 |
| 瓶颈标识 | AI标注最慢链路 |
| 各节点QPS | 表格：接口名/QPS/RT/状态 |

## 七、报告导出

| 格式 | 内容 | 触发方式 |
|------|------|---------|
| **PDF** | 完整报告（图表+表格+截图） | 手动导出 / 定时任务 |
| **HTML** | Web版报告（可分享链接） | 自动生成 |
| **邮件** | 摘要+附件PDF | 定时任务(每日/每周) |

定时报告：每天早9点推送昨日质量日报；每周一推送上周质量周报。
