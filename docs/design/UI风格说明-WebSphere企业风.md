# UI风格说明：WebSphere企业风

## 当前原型采用的UI风格

我们采用的是 **WebSphere企业风**（也称 IBM Carbon Design System 风格），这是企业级B端产品的黄金标准。

---

## WebSphere企业风的核心特点

### 1. 色彩体系
```
主色调：深藏蓝 #0F1419（侧边栏、顶部导航）
辅助色：专业蓝 #0F62FE（按钮、链接、选中态）
背景色：浅灰白 #F4F7FB（页面背景）
卡片色：纯白色 #FFFFFF（数据卡片）
数据色：翡翠绿 #198038（成功/正常）
       琥珀黄 #F1C21B（警告/注意）
       信号红 #DA1E28（错误/异常）
```

**为什么用这个配色？**
- 深蓝给人"专业、可信、稳定"的感觉，适合测试平台这种严肃工具
- 不是纯黑(#000)，而是深藏蓝(#0F1419)，减少视觉疲劳
- 数据色遵循"绿色=好，红色=坏"的直觉，不需要学习成本

---

### 2. 布局结构
```
┌─────────────────────────────────────────────┐
│  顶部导航栏（深蓝，48px高，白色文字）        │  ← 全局导航
├──────────┬──────────────────────────────────┤
│          │  页面标题 + 操作按钮               │
│  侧边栏   │  ───────────────────────────────  │
│  （深蓝   │  筛选栏（边框/背景区分）           │
│  240px） │  ───────────────────────────────  │
│          │  数据卡片区（3-4列，有间隔）        │
│  菜单项   │  ───────────────────────────────  │
│  图标+   │  数据表格（斑马纹，悬停高亮）       │
│  文字    │  ───────────────────────────────  │
│          │  分页器 + 统计信息                 │
└──────────┴──────────────────────────────────┘
```

**为什么是这个布局？**
- 侧边栏固定，内容区滚动——用户永远知道自己在哪个模块
- 卡片+表格混合布局——概览用卡片（一眼看懂），详情用表格（精确查找）
- 筛选栏始终在内容区顶部——不需要滚走才能筛选

---

### 3. 数据密度
WebSphere风格的核心是 **"高数据密度但不拥挤"**：

| 设计元素 | 常见错误 | WebSphere做法 |
|---------|---------|--------------|
| 卡片间距 | 16px（太松，一屏看不了几个） | 20px（紧凑但透气） |
| 表格行高 | 32px（太挤） | 48px（舒适阅读） |
| 字号 | 全文14px | 标题16px/正文14px/辅助12px |
| 边框 | 全边框（视觉噪点） | 底部边框+悬停高亮 |

---

### 4. 交互细节
```
按钮：
  - 主按钮：实心蓝底白字（关键操作）
  - 次级按钮：蓝边框蓝字（普通操作）
  - 危险按钮：红边框红字（删除/停止）

表格：
  - 表头：浅灰底(#F7F9FB) + 加粗
  - 行悬停：浅蓝底(#EDF5FF)
  - 选中行：蓝底白字

状态标签：
  - 成功：绿底绿字
  - 失败：红底红字
  - 进行中：蓝底蓝字 + 旋转动画
```

### 5. 与主流风格对比

| 风格 | 代表 | 适合场景 | 不适合我们原因 |
|------|------|---------|--------------|
| **Ant Design默认** | 蚂蚁金服 | 中后台系统 | 太通用，没有测试平台的"专业感" |
| **Material Design** | Google | 移动端/C端 | 圆角太多，不适合数据密集场景 |
| **Tailwind默认** | 开发者友好 | 快速原型 | 需要大量定制才能达到企业级质感 |
| **✅ WebSphere** | IBM/企业级SaaS | **B端数据平台** | **最适合测试平台** |

## 为什么WebSphere风适合测试平台？

### 理由1：测试人员的工作场景
测试人员每天看的是：
- 大量测试结果（通过/失败）
- 执行日志（堆栈信息）
- 趋势图表（RT/QPS变化）

WebSphere的高数据密度设计，让一屏能展示更多信息，减少滚动。

### 理由2：企业采购决策
如果是企业版（付费），采购决策者是CTO/测试经理，他们看到WebSphere风格会想：
> "这个工具看起来很专业，像IBM/微软的产品，值得信赖"

### 理由3：国际化预留
WebSphere风格是国际标准，后期做英文版时不需要改设计，只需要替换文字。

## 确认问题

在开始出原型之前，确认几个细节：

1. **APP自动化优先还是WEB自动化优先？**
   - 建议：先WEB（Playwright），因为更稳定；APP（Appium）作为补充

2. **PO设计模式的语言？**
   - Python（与后端一致）还是 Java（MeterSphere用Java）？
   - 建议：Python，因为AI生成代码更友好

3. **元素定位策略优先级？**
   - 推荐：ID > CSS > XPath > 图像识别（Airtest）
   - 是否需要"智能定位"（自动选择最稳定的策略）？

请确认，我马上开始出原型和详细技术方案！
