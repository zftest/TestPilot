## 一、任务执行全流程

```
触发执行（手动/CI/定时/Webhook）
        ↓
  ┌─ 执行队列（Redis List）─────┐
  │  根据优先级排序               │
  │  高优先级 -> 立即分配         │
  │  普通优先级 -> FIFO排队       │
  └──────────┬──────────────────┘
             ↓
  ┌─ 调度器（Celery Beat）──────┐
  │  检查可用执行节点             │
  │  分配任务到Worker             │
  │  发送开始通知                 │
  └──────────┬──────────────────┘
             ↓
  ┌─ Worker执行（独立进程）─────┐
  │  加载用例 → 执行 → 采集结果  │
  │  实时WebSocket推送进度       │
  │  失败自动重试（最多3次）     │
  │  生成截图/录屏               │
  └──────────┬──────────────────┘
             ↓
  ┌─ 结果聚合 ─────────────────┐
  │  Celery任务聚合所有Worker结果│
  │  写入PostgreSQL分析库        │
  │  触发AI失败分析              │
  │  触发质量门禁检查            │
  │  生成报告HTML/PDF            │
  │  发送通知（飞书/企微/邮件）   │
  └─────────────────────────────┘
```

## 二、执行队列管理（UI原型要点）

### 2.1 队列列表页

| 字段 | 说明 |
|------|------|
| 执行ID | 唯一标识 |
| 触发方式 | 手动/CI/CD/定时/Webhook（带图标） |
| 测试计划 | 计划的名称 |
| 优先级 | 高/中/低 |
| 排队时间 | 开始排队的时间 |
| 状态 | 排队中/分配中/执行中/等待重试/已完成/失败/已取消 |
| 操作 | 取消执行/调整优先级/查看详情 |

### 2.2 队列操作

- **取消**：取消排队中的任务（执行中的不允许取消，只能终止）
- **调优先级**：管理员可将某任务优先级调高
- **并发控制**：全局最大并发执行数可配置（默认5个并行任务）
- **节点管理**：查看各Worker节点状态（在线/离线/负载）

## 三、实时执行监控页

### 3.1 监控面板布局

| 区域 | 内容 |
|------|------|
| **顶部状态栏** | 执行ID / 进度条(%+已执行/总数) / 耗时计时器 / 状态标签 |
| **左侧(60%)** | 用例执行状态流（滚动列表），每条用例：名称+状态图标+耗时 |
| **右侧(40%)** | 实时日志流 / 失败详情（点击失败用例时展示） |
| **底部** | 暂停/继续/终止 按钮 |

### 3.2 用例状态流转

```
pending → running → passed
                  → failed → retrying → passed
                                     → failed(最终)
                  → skipped
                  → blocked
```

### 3.3 WebSocket实时数据

| 推送事件 | 频率 | 数据 |
|---------|------|------|
| execution.progress | 每秒 | {completed, total, passed, failed} |
| case.result | 每条用例完成时 | {case_id, status, duration, error_msg} |
| execution.log | 实时 | {level, message, timestamp} |
| execution.complete | 执行完成时 | {summary, report_url} |

## 四、报告查看入口与导航

### 4.1 入口路径

| 入口 | 路径 | 场景 |
|------|------|------|
| **执行列表** | 测试执行 > 执行记录 > 点击某条记录 | 日常查看 |
| **项目仪表盘** | 工作台 > 最近执行 > 点击 | 快速跳转 |
| **CI/CD回调** | Pipeline详情页 > 报告链接 | 开发/DevOps |
| **飞书/企微通知** | 消息卡片中的链接 | 移动端查看 |
| **报告列表** | 测试报告 > 执行报告 > 搜索 | 历史查找 |

### 4.2 报告页导航结构

```
执行报告详情页
├── 顶部：执行摘要（通过率/耗时/触发方式/时间）
├── Tab导航
│   ├── 📊 概览      ← 环形图+关键指标卡片
│   ├── 📋 用例详情   ← 可筛选/排序的用例表格
│   ├── 🐛 失败分析   ← AI诊断+失败截图
│   ├── 📈 趋势对比   ← 与历史执行对比
│   └── 📜 日志      ← 完整执行日志
├── 底部操作栏
│   ├── 导出PDF / 分享链接 / 重新执行 / 创建缺陷
```

## 五、历史报告对比

### 5.1 对比模式

选择2-5条历史执行记录，并排对比：

| 对比维度 | 展示方式 |
|---------|---------|
| 通过率变化 | 折线图（选中点高亮） |
| 新增失败用例 | 差异列表（标记NEW） |
| 修复的用例 | 差异列表（标记FIXED） |
| 执行耗时变化 | 柱状图对比 |

### 5.2 对比入口

执行报告 > 趋势对比 Tab > 选择基准版本 > 自动生成对比报告

## 六、定时报告

### 6.1 配置项

| 配置 | 选项 |
|------|------|
| 频率 | 每天/每周/每两周/每月 |
| 时间 | 指定发送时间 |
| 范围 | 全部项目/指定项目 |
| 收件人 | 邮件列表/飞书群/企微群 |
| 格式 | PDF(附件) + HTML(链接) |

### 6.2 报告模板

- **日报**：昨日执行摘要 + 失败TOP5 + 缺陷趋势
- **周报**：本周通过率趋势图 + 缺陷看板截图 + 下周计划
- **月报**：月度质量报告 + 自动化覆盖率 + AI使用统计
