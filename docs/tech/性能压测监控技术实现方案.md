## 二、实时数据采集方案

### 2.1 单接口压测数据采集

**采集点：** Locust Runner内部钩子

```python
# perf_collector.py
from locust import events
from datetime import datetime
import json

@events.request.add_listener
def on_request(request_type, name, response_time, exception, **kwargs):
    """每次HTTP请求完成后触发"""
    metric = {
        "timestamp": datetime.utcnow().isoformat(),
        "endpoint": name,
        "response_time_ms": response_time,
        "exception": str(exception) if exception else None,
        "status_code": kwargs.get("context", {}).get("response", {}).get("status_code")
    }
    # 写入Redis Stream（高性能缓冲）
    redis_client.xadd("perf:raw:{scenario_id}", metric)
```

**聚合计算（每5秒）：**
```python
# 使用Redis + Celery异步聚合
def aggregate_metrics(scenario_id, window_seconds=5):
    """滑动窗口聚合：QPS、平均RT、P95、P99、错误率"""
    raw_key = f"perf:raw:{scenario_id}"
    # 从Redis Stream读取最近5秒数据
    entries = redis_client.xrange(raw_key, min='-', max='+', count=10000)
    
    # 计算聚合指标
    response_times = [float(e[1]['response_time_ms']) for e in entries]
    errors = [e for e in entries if e[1]['exception']]
    
    return {
        "qps": len(entries) / window_seconds,
        "avg_rt": statistics.mean(response_times),
        "p95_rt": numpy.percentile(response_times, 95),
        "p99_rt": numpy.percentile(response_times, 99),
        "error_rate": len(errors) / len(entries) if entries else 0
    }
```

## 三、实时推送方案（WebSocket）

### 3.1 后端WebSocket推送（FastAPI + Socket.IO）

```python
# ws_perf.py
from fastapi import APIRouter, WebSocket, WebSocketDisconnect
from typing import Dict, List
import json
import asyncio

router = APIRouter()
active_connections: Dict[int, List[WebSocket]] = {}  # perf_run_id → [ws1, ws2]

@router.websocket("/ws/perf/{perf_run_id}")
async def perf_metrics_ws(websocket: WebSocket, perf_run_id: int):
    await websocket.accept()
    # 注册连接
    if perf_run_id not in active_connections:
        active_connections[perf_run_id] = []
    active_connections[perf_run_id].append(websocket)
    
    try:
        # 每1秒推送一次聚合数据
        while True:
            metrics = get_latest_aggregated_metrics(perf_run_id)
            await websocket.send_json({
                "type": "perf_update",
                "perf_run_id": perf_run_id,
                "timestamp": datetime.utcnow().isoformat(),
                "data": metrics
            })
            await asyncio.sleep(1)
    except WebSocketDisconnect:
        active_connections[perf_run_id].remove(websocket)
```

### 3.2 前端WebSocket消费

```javascript
// PerfMonitor.tsx
import { io } from 'socket.io-client';

const PerfMonitor = ({ perfRunId }) => {
  const [metrics, setMetrics] = useState(null);
  const chartRef = useRef(null);

  useEffect(() => {
    const socket = io('/api/v1');
    socket.emit('join_perf_run', perfRunId);
    
    socket.on('perf_update', (data) => {
      setMetrics(data.data);
      // 实时更新Chart.js图表（不重绘，只更新数据）
      if (chartRef.current) {
        const chart = chartRef.current;
        chart.data.datasets[0].data.push(data.data.current.qps);
        if (chart.data.datasets[0].data.length > 60) chart.data.datasets[0].data.shift();
        chart.update('none');  // 无动画更新，保证流畅
      }
    });

    return () => socket.disconnect();
  }, [perfRunId]);

  return <div>...</div>;
};
```

## 四、数据存储方案

### 4.1 时序数据表（TimescaleDB超表）

```sql
-- 启用TimescaleDB扩展
CREATE EXTENSION IF NOT EXISTS timescaledb;

-- 性能指标时序表（超表）
CREATE TABLE perf_metrics (
  time        TIMESTAMPTZ NOT NULL,
  perf_run_id BIGINT       NOT NULL,
  scenario_id BIGINT       NOT NULL,
  metric_name  VARCHAR(64)  NOT NULL,  -- qps/avg_rt/p95_rt/error_rate/cpu/mem/disk/net
  metric_value FLOAT        NOT NULL,
  node_name   VARCHAR(128),
  labels_json JSONB        DEFAULT '{}',
  PRIMARY KEY (time, perf_run_id)
);

-- 转换为超表（按时间自动分区）
SELECT create_hypertable('perf_metrics', 'time', chunk_time_interval => INTERVAL '1 hour');

-- 压缩策略（历史数据自动压缩）
SELECT add_compression_policy('perf_metrics', INTERVAL '7 days');
```

### 4.2 调用链数据表

```sql
CREATE TABLE trace_spans (
  trace_id      VARCHAR(64)  NOT NULL,
  span_id       VARCHAR(64)  NOT NULL,
  parent_span_id VARCHAR(64),
  operation_name VARCHAR(255) NOT NULL,
  start_time    TIMESTAMPTZ NOT NULL,
  duration_ms   INT          NOT NULL,
  perf_run_id   BIGINT       NOT NULL,
  tags_json      JSONB         DEFAULT '{}',
  PRIMARY KEY (trace_id, span_id)
);

CREATE INDEX idx_trace_perf_run ON trace_spans(perf_run_id, start_time);
```

## 六、部署架构

### 6.1 Docker Compose全栈部署

```yaml
# docker-compose.perf.yml
version: '3.8'
services:
  # Locust Master（压测控制器）
  locust-master:
    image: locustio/locust:2.24
    command: -f /scripts/perf_scenario.py --master --expect-workers=3
    volumes: ["./perf_scripts:/scripts"]
    ports: ["8089:8089"]
    environment:
      - LOCUST_MODE=master
  
  # Locust Workers（压测执行节点，可扩展）
  locust-worker:
    image: locustio/locust:2.24
    command: -f /scripts/perf_scenario.py --worker --master-host=locust-master
    deploy:
      replicas: 3  # 3个Worker节点
    volumes: ["./perf_scripts:/scripts"]
  
  # Prometheus（指标采集）
  prometheus:
    image: prom/prometheus:latest
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.retention.time=30d'
    volumes: ["./prometheus.yml:/etc/prometheus/prometheus.yml"]
    ports: ["9090:9090"]
  
  # Redis（实时数据缓冲）
  redis:
    image: redis:7-alpine
    command: redis-server --maxmemory 512mb --maxmemory-policy allkeys-lru
    ports: ["6379:6379"]
  
  # TimescaleDB（时序数据存储）
  timescaledb:
    image: timescale/timescaledb:latest-pg14
    environment:
      - POSTGRES_PASSWORD=testops123
    volumes: ["./pgdata:/var/lib/postgresql/data"]
```

## 七、性能优化要点

| 优化点 | 方案 | 效果 |
|--------|------|------|
| WebSocket连接数 | Redis Pub/Sub分发，支持水平扩展 | 支持1000+并发观察者 |
| 前端图表渲染 | Chart.js `update('none')` 无动画更新 | 60FPS实时流畅 |
| 时序数据写入 | 批量写入（1000条/批）+ TimescaleDB | 10万指标/秒写入 |
| 历史数据查询 | 连续聚合（Continuous Aggregate） | 30天数据查询 < 200ms |
| 告警计算 | 后台Celery任务，每10秒计算一次 | 实时告警延迟 < 15s |
