## 二、安装清单（每台被监控服务器必须安装）

### 2.1 Node Exporter（必装，采集系统级指标）

```bash
# 下载
wget https://github.com/prometheus/node_exporter/releases/download/v1.7.0/node_exporter-1.7.0.linux-amd64.tar.gz
tar xzf node_exporter-1.7.0.linux-amd64.tar.gz
cd node_exporter-1.7.0.linux-amd64

# 安装为systemd服务
sudo cp node_exporter /usr/local/bin/
sudo tee /etc/systemd/system/node_exporter.service <<EOF
[Unit]
Description=Prometheus Node Exporter
After=network.target

[Service]
User=nobody
ExecStart=/usr/local/bin/node_exporter \
  --collector.cpu \
  --collector.meminfo \
  --collector.diskstats \
  --collector.netdev \
  --collector.netstat \
  --collector.loadavg \
  --collector.filesystem

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable node_exporter
sudo systemctl start node_exporter
```

### 2.2 采集的核心指标

| 指标类别 | 指标名 | 说明 |
|---------|--------|------|
| CPU | node_cpu_seconds_total | CPU各模式时间 |
| 内存 | node_memory_MemAvailable_bytes | 可用内存 |
| 磁盘 | node_filesystem_avail_bytes | 磁盘可用空间 |
| 磁盘IO | node_disk_read_bytes_total | 磁盘读写速率 |
| 网络 | node_network_receive_bytes_total | 网络接收流量 |
| 网络连接 | node_netstat_Tcp_CurrEstab | TCP当前连接数 |
| 负载 | node_load1 / load5 / load15 | 系统负载 |

### 2.3 可选Exporter

| Exporter | 端口 | 用途 | 安装场景 |
|----------|------|------|---------|
| **MySQL Exporter** | 9104 | MySQL慢查询/连接数/QPS | 自建MySQL需监控 |
| **PostgreSQL Exporter** | 9187 | PG连接数/查询耗时/缓存命中率 | 自建PG需监控 |
| **Redis Exporter** | 9121 | Redis内存/命中率/连接数 | 使用Redis需监控 |
| **cAdvisor** | 8080 | Docker容器CPU/内存/网络 | Docker化部署推荐 |
| **Blackbox Exporter** | 9115 | HTTP/TCP/DNS探测 | 外部可用性监控 |

## 三、Docker Compose 监控栈（一键部署）

```yaml
# docker-compose.monitoring.yml
version: '3.8'

services:
  prometheus:
    image: prom/prometheus:v2.50.0
    container_name: prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
      - prometheus_data:/prometheus
      - ./alert.rules.yml:/etc/prometheus/alert.rules.yml
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
      - '--storage.tsdb.retention.time=30d'
      - '--web.enable-lifecycle'
    restart: always

  alertmanager:
    image: prom/alertmanager:v0.27.0
    container_name: alertmanager
    ports:
      - "9093:9093"
    volumes:
      - ./alertmanager.yml:/etc/alertmanager/alertmanager.yml
    restart: always

  grafana:
    image: grafana/grafana:10.3.0
    container_name: grafana

      - "3000:3000"
    environment:
      - GF_SECURITY_ADMIN_USER=admin
      - GF_SECURITY_ADMIN_PASSWORD=${GRAFANA_PASSWORD}
      - GF_USERS_ALLOW_SIGN_UP=false

      - grafana_data:/var/lib/grafana
      - ./grafana/dashboards:/etc/grafana/provisioning/dashboards
      - ./grafana/datasources.yml:/etc/grafana/provisioning/datasources/datasources.yml
    restart: always

volumes:
  prometheus_data:
  grafana_data:
```

## 四、Prometheus 配置详解

### 4.1 prometheus.yml

```yaml
global:
  scrape_interval: 15s
  evaluation_interval: 15s
  external_labels:
    cluster: 'testpilot-prod'

alerting:
  alertmanagers:
    - static_configs:
        - targets: ['alertmanager:9093']

rule_files:
  - '/etc/prometheus/alert.rules.yml'

scrape_configs:
  # 自身监控
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  # 应用服务器
  - job_name: 'testpilot-app-servers'
    static_configs:
      - targets:
          - '192.168.1.10:9100'   # 应用服务器1
          - '192.168.1.11:9100'   # 应用服务器2
          - '192.168.1.12:9100'   # 应用服务器3
        labels:
          env: 'production'
          role: 'app'

  # 数据库服务器
  - job_name: 'mysql'
    static_configs:
      - targets:
          - '192.168.1.20:9104'
        labels:
          env: 'production'
          role: 'database'

  - job_name: 'postgresql'
    static_configs:
      - targets:
          - '192.168.1.20:9187'
        labels:
          env: 'production'
          role: 'database'

  # Docker容器（通过cAdvisor）
  - job_name: 'docker'
    static_configs:
      - targets:
          - '192.168.1.10:8080'
          - '192.168.1.11:8080'

  # Redis
  - job_name: 'redis'
    static_configs:
      - targets:
          - '192.168.1.30:9121'

  # 外部可用性探测
  - job_name: 'blackbox'
    metrics_path: /probe
    params:
      module: [http_2xx]
    static_configs:
      - targets:
          - 'https://testpilot.example.com/health'
    relabel_configs:
      - source_labels: [__address__]
        target_label: __param_target
      - source_labels: [__param_target]
        target_label: instance
      - target_label: __address__
        replacement: blackbox:9115
```

## 五、告警规则（alert.rules.yml）

```yaml
groups:
  - name: server_alerts
    rules:
      # CPU使用率 > 90%持续5分钟
      - alert: HighCPUUsage
        expr: 100 - (avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 90
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "服务器 {{ $labels.instance }} CPU使用率超过90%"
          description: "当前值: {{ $value }}%"

      # 内存可用 < 10%
      - alert: LowMemory
        expr: (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes) * 100 < 10
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "服务器 {{ $labels.instance }} 内存不足"
          description: "可用内存仅 {{ $value }}%"

      # 磁盘可用 < 20%
      - alert: LowDiskSpace
        expr: (node_filesystem_avail_bytes{fstype!="tmpfs"} / node_filesystem_size_bytes) * 100 < 20
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "服务器 {{ $labels.instance }} 磁盘空间不足"
          description: "挂载点 {{ $labels.mountpoint }}，可用 {{ $value }}%"

      # 磁盘可用 < 5%（紧急）
      - alert: CriticalDiskSpace
        expr: (node_filesystem_avail_bytes{fstype!="tmpfs"} / node_filesystem_size_bytes) * 100 < 5
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "服务器 {{ $labels.instance }} 磁盘空间严重不足！"
          description: "挂载点 {{ $labels.mountpoint }}，仅剩 {{ $value }}%"

      # MySQL连接数 > 80%
      - alert: MySQLHighConnections
        expr: mysql_global_status_threads_connected / mysql_global_variables_max_connections * 100 > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "MySQL连接数超过80%"

      # Redis内存 > 80%
      - alert: RedisHighMemory
        expr: redis_memory_used_bytes / redis_total_system_memory_bytes * 100 > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Redis内存使用超过80%"

      # 服务器宕机
      - alert: ServerDown
        expr: up == 0
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "服务器 {{ $labels.instance }} 不可达！"
          description: "已超过2分钟无法连接"
```

## 七、TestPilot平台内嵌监控

### 7.1 集成方式

Grafana支持匿名访问 + iframe嵌入，TestPilot前端通过 `<iframe>` 嵌入Grafana Dashboard，实现统一平台内展示。

```javascript
// Grafana数据源配置（datasources.yml）
apiVersion: 1
datasources:
  - name: Prometheus
    type: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: true
```

### 7.2 三大监控大盘对应关系

| TestPilot大盘 | Grafana Dashboard | 核心面板 |
|--------------|-------------------|---------|
| 单接口压测 | Locust Dashboard (内建) | Locust自身stats |
| 链路压测 | Custom Trace Dashboard | Jaeger数据源 |
| 服务器资源 | Node Exporter Full Dashboard | Prometheus数据源 |

### 7.3 服务器资源大盘预置面板（Node Exporter Full）

| 行 | 面板内容 |
|----|---------|
| 第1行 | CPU使用率 / 负载 / 内存 / 磁盘 4个stat面板 |
| 第2行 | CPU详细（user/system/iowait/steal）折线图 |
| 第3行 | 内存详细（used/cached/buffer/swap）面积图 |
| 第4行 | 磁盘IO（读写速率+IOPS）折线图 |
| 第5行 | 网络流量（进/出）折线图 |
| 第6行 | 服务器列表（全部节点CPU/内存/磁盘表格） |
| 第7行 | 告警面板（当前活跃告警） |

## 八、压测时服务器资源关联

### 8.1 压测→资源联动

当发起性能压测时，TestPilot自动记录压测开始/结束时间戳，前端展示服务器资源面板时自动筛选对应时间范围，实现压测期间资源消耗的精准展示。

### 8.2 压测报告中的资源分析

每条压测报告自动附带：
- 压测期间的CPU峰值/均值/谷值
- 压测期间的内存峰值/均值
- 压测期间的磁盘IO峰值
- 压测期间的网络流量总计
- 资源瓶颈标注（如 CPU达到95%时QPS开始下降）
