## 一、产品定位

TestPilot 是一个 **AI-Native 全栈自动化测试平台**，覆盖 API测试、Web UI测试、APP UI测试、性能压测 四大测试类型，以 AI 作为操作系统的核心。

**核心理念：** 不是给传统测试平台加一个 AI 按钮，而是 AI 本身就是操作系统。

**一句定位：** 测试团队的 AI 副驾驶——从需求分析到用例执行再到报告输出，全程 AI 驱动。

---

## 二、目标市场与用户

| 用户角色 | 核心痛点 | TestPilot 价值 |
|----------|---------|---------------|
| 测试工程师 | 手动写用例效率低、定位变更维护工作量大 | AI自然语言生成用例 + 自愈 |
| 测试Leader | 平台重(部署复杂)、报表不直观 | Docker一键部署、实时监控大盘 |
| 开发工程师 | 自测缺乏好用的工具 | CI/CD插件一键触发、Git联动 |
| PM/QA Manager | 需求覆盖度不可见、发版决策缺数据 | 需求追溯矩阵、质量门禁 |

### 3.4 CI/CD集成方案

| CI工具 | 集成方式 | 提供内容 |
|--------|---------|---------|
| Jenkins | Pipeline Script (Jenkinsfile) | 完整Groovy脚本：触发→轮询→门禁→部署 |
| GitLab CI | .gitlab-ci.yml | 完整YAML：4个stage(trigger/wait/gate/deploy) |
| GitHub Actions | workflow .yml | 完整YAML：on push/PR触发+门禁判定 |

统一流程：Git Push → Pipeline触发 → TestPilot API执行测试 → 轮询等待 → 质量门禁检查 → 通过/阻塞

### 3.5 服务器监控方案

- **被监控端**：Node Exporter(必装) + MySQL Exporter/PG Exporter/Redis Exporter/cAdvisor(按需)
- **采集存储**：Prometheus Server，15秒采集，TSDB存储30天
- **告警路由**：AlertManager → 飞书/企微/钉钉/邮件
- **可视化**：Grafana Dashboard iframe嵌入TestPilot平台
- **Docker Compose**：prometheus + alertmanager + grafana 三容器一键部署

## 四、权限模型

| 角色 | 权限范围 |
|------|---------|
| super_admin | 全部权限，含系统配置/用户管理/许可证 |
| admin | 团队+项目管理+所有测试功能+报告 |
| pm | 项目管理+报告+需求+质量门禁配置 |
| tester | 用例设计+执行+报告查看+缺陷提交 |
| viewer | 只读（查看报告和趋势） |

权限粒度：资源(resource) × 操作(action)，共17项权限，支持角色-权限树勾选配置。
